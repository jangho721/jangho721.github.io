Construction Site Hazard Detection

A study on using image augmentation to recognize small hazard objects in indoor construction sites — measuring which augmentation strategies actually improve a CNN's accuracy when real images are scarce. Published as a co-authored paper.

Role
Undergraduate Researcher · Construction Systems Lab, Inha University
Timeline
2019 – 2020
Stack
Python · CNN · Computer Vision
Focus
Image recognition · Data augmentation · Data-centric ML

01. Problem

Indoor construction sites are littered with small tools — hammers, shovels, electric drills, spanners — that get left lying around and cause accidents. The lab's broader goal was to automate safety-rule checking, which first requires recognizing these objects from site imagery.

Recognition is data-hungry, but labeled images of these small objects are scarce, so a model trained on them doesn't generalize well. The question I focused on: can image augmentation close that data gap — and which augmentations actually help?

02. What I did

  1. A 4-class recognition CNN

    Built a convolutional network (224×224 input → stacked convolution / batch-norm / max-pool blocks → fully-connected layers → softmax) to recognize the four hazard objects from a base set of 4,000 images, 1,000 per class.

  2. Augmentation to beat data scarcity

    Generated more training data with four augmentations — shift (≤20%), rotation (≤30°), zoom (0.5–1.5×), and brightness (0.5–1.5×) — combined two at a time into six training sets (A–F).

  3. Measured what actually helps

    Trained the same CNN on each augmented set and compared recognition accuracy against the un-augmented baseline, to see which augmentations are worth using for small construction-site objects.

03. Results

Training set Accuracy
Baseline (none) 85.7%
A · rotation + shift 85.1%
B · rotation + brightness 86.9%
C · brightness + shift 85.8%
D · rotation + zoom 85.7%
E · shift + zoom 87.3%
F · brightness + zoom 87.6%
  • Augmentation lifted accuracy by +0.7% on average, up to +1.9% — modest, but consistent where it counted. Zoom-based sets (E, F) helped most.
  • Rotation-heavy sets (A, D) didn't reliably improve, suggesting rotation is a poor fit for these small, orientation-stable objects — a more useful takeaway than the headline number.

04. Context

This was my first hands-on machine-learning research, during an undergraduate internship at the Construction Systems Laboratory. The methods are simple, but it's where I learned to set up a clean experiment, read what the numbers were really saying, and see a model aimed at a real, physical problem — and it's what got me to pursue ML seriously.
